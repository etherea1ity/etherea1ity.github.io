第九章无线网络和移动网络

重要内容

  • CSMA/CA

核心知识点

无线局域网WLAN

  1. WLAN（Wireless Local Area Network）
  2. 便携站（portable station）和移动站（mobile station），移动站不仅能移动，且能在移动中通信

无线局域网的组成

两类 WLAN,AdHoc，无线传感器网络，CSMA/CA,802.11 帧（四个地址）

两类 WLAN：

  1. 有固定基础设施的 WLAN 主机之间的通信必须通过 AP 的转发
  2. 无固定基础设施的 WLAN（移动自组织网络 Adhoc） 无 AP，临时网络，不和其他网络连接；结点需运行路由选择协议，结点需转发分组
IEEE 802.11
  1. 无线局域网的中心叫做接入点AP（Access Point）。
  2. 802.11标准规定无线局域网的最小构件是基本服务集BSS（Basic Service Set），一个基本服务集BSS包括（至少）一个接入点和若干个移动站。各站在本站内的通信或与外部站点的通信，都要通过本BSS的接入点（基站）。
  3. 安装AP时，需要分配一个不超过32字节的服务集标识符SSID（Service Set Identifier）和一个通信信道，SSID是其名字，为字符串而不是二进制。
  4. 一个BSS覆盖的地理范围叫一个基本服务区BSA（Basic Service Area）。
  5. AP出场时就有一个唯一的48位2进制数字的MAC地址，称为基本服务集标识BSSID。一个BSS可以通过接入点AP连接到一个分配系统DS（Distribution System），再连接到另一个基本服务集，成为一个扩展服务集ESS（Extended Service System），ESS也有标识符，为不超过32字符的字符串名字而不是地址，称为扩展服务集标识符ESSID。
  6. ESS还可以通过门户（portal）为无限用户提供到802.x的局域网（也就是非802.11的），作用相当于一个网桥。

  1. 漫游中，与接入点AP建立关联（association），分为主动和被动扫描
  2. 被动扫描
    1. 接入点AP周期性发出信标帧（beacon frame）
    2. 移动站A扫描11个信道，选择愿意加入接入点的AP所在的基本服务集BSS2，于是向AP发出关联请求帧（association request frame）
    3. 接入点AP同意移动站A发来的关联请求，向移动站A发送关联响应帧（association response frame）
  3. 主动扫描
    1. 移动站A主动发出探测请求帧（probe request frame）
    2. 接入点AP回答探测响应帧（probe response frame）
    3. 移动站A向AP发出关联请求帧
    4. AP回应关联响应帧，建立连接
移动自组网络
  • 自组网络是没有固定基础设施的(即没有AP)的无线局域网。这种网络由一些处于平等状态的移动站之间相互通信组成的临时网络。

  • 无线传感器网络 WSN： 低功耗、低宽带、低存储容量、物联网

  • 一些名词

    • 固定接入（fixed access）：接入期间用户地理位置不变
    • 移动接入（mobility access）：接入期间以车速移动
    • 便携接入（portable access）：接入期间步行移动
    • 游牧接入（nomadic access）：接入期间至少在通信的时候不变，有时和便携接入一样

802.11 局域网的物理层

802.11 局域网的物理层

CSMA/CD载波侦听多路访问/冲突检测协议（Carrier Sense Multiple Access/ Collision Detection）
  1. 不能用有线局域网的CSMA/CD协议：无线信道的信号强度的动态范围非常大，802.11适配器上接收到的信号强度往往远远小于发送信号的强度，因此无法实现碰撞检测。
  2. 隐蔽站问题（hidden station）：站点A和C都想向B通信，但AC互相检测不到对方（距离太远或者障碍物），因此发生了碰撞，并且无法检测出。

CSMA/CA碰撞避免（Collision Avoidance）
  1. 链路层确认：使用停止等待协议，发送完一帧后等到收到确认帧才继续发送。
  2. 802.11MAC层通过协调功能（Coordination Function）来确定在基本服务集BSS中的移动站发送/接收数据的时间。
  3. ==必选==分布协调功能（Distributed Coordination Function）：各个站点通过争用信道获得使用权，有短帧间间隔SIFS（Short Inter-Frame Spacing）和分布协调功能帧间间隔（DCF IFS）
  4. ==可选==点协调功能PCF

CSMA/CA原理

  • 基本原理：如果在DIFS时间内检测到信道闲，站点A可以发送数据帧DATA，B收到后返回确认ACK，发送确认的时刻为接收到数据后的一个SIFS（因为B需要CRC检验）。所以在DATA+SIFS+ACK时间内，一直信道忙。
虚拟载波监听（Virtual Carrier Sense）
  • 假如A要发送数据给B，把源站A要占用的时间（Data+ACK+SIFS）写入数据帧DATA的首部，通知给其他处在站点A广播范围的站，各个站建立自己的网络分配向量NAV（Network Allocation Vector），期间不能发送数据。
载波监听/随机退避
  • 退避算法

  • 如果站点AB通信时，CDE也想发送数据，由于检测到信道忙，需要推迟接入（defer access），所有推迟接入的站点必须在争用期执行统一的退避算法公平地争用信道。

  1. 争用窗口CW（Contention Window）：由许多时隙（time slot）组成，802.11g规定一个时隙为9us，SIFS=10us，而DIFS应比SIFS多两个时隙，为28us。

  2. 退避计时器（backoff timer）：上图原理用来记录退避时间的。

  3. 退避算法：站点进入争用期，应在0~CW个时隙中随机生成一个退避时隙并设置退避计时器。计时器最先降为0的站最先接入媒体发送数据帧。
  4. 二进制指数退避：如争用期为$2^4-1=15$，那么第i次退避的时隙数应在$0到2^{4+i}-1$中生成。

  5. 若检测信道连续空闲时间超过DIFS时，可立即发送数据，而不必经过争用期。

  6. 必须经过争用期：

    1. 发送数据时信道忙
    2. 发送的数据帧未收到确认重传
    3. 接着发送后续的数据帧（发送成功后）

  7. 缺点：不能高效解决隐蔽站/暴露站问题

预约信道

若站点A要向站点B发送数据帧，经过AP，有一个隐蔽站C

  1. A在向AP发送数据帧DATA前，先发送一个很短的控制帧发送请求RTS（Request To Send），占用时间[SIFS+CTS+SIFS+DATA+SIFS+ACK]，写在RTS首部，只有B站能收到。
  2. AP接收到RTS后，经过一个SIFS，向A发送一个允许发送CTS（Clear To Send），占用时间[SIFS+DATA+SIFS+SCK]，A、B和C站都能收到
  3. A站发送的数据帧首部写入时间[DATA+SIFS+SCK]

  1. 优缺点：解决了隐蔽站的问题，但是如果有的站在RTS发送前后发送数据帧，那么还是会发生碰撞。

802.11 局域网的MAC帧

  • 802.11帧有三种类型：控制帧、数据帧(0000)和管理帧。

  • 数据帧三大部分：
    • MAC首部30字节
    • 帧主体不超过2312字节，通常小于1500字节
    • 帧检验序列FCS，尾部，4字节
802.11 数据帧地址
  • 地址1、2分别是接收地址和发送地址，3、4取决于数据帧中的来自、去往AP。
  • 其中AP的MAC地址就是BSSID。

序号控制、持续期、帧控制字段
  • 功率管理字段只有1位，用来指示移动站的功率管理模式。

蜂窝移动通信网

3G

引入了无线接入网的概念，用户设备UE（User Equipment）和基站NB之间是无线线路。

使用 IP 的体系结构和混合的交换机制，3G开始以后的各代蜂窝移动通信都是以传输数据业务为主的通信系统

4G

  • 取消了电路交换，无论数据还是语音都是用分组交换，或称为全网IP化。
  • 发布了LTE（Long Term Evolution）标准，由三大部分组成，用户设备UE，演进的无线接入网E-UTRAN（Evolved UTRAN）和演进的分组核心网EPC（Evolved Packet Core）。EPC上半部分是控制层面（传输信令），下半部分是用户层面（传输数据）。

移动IP

  • 移动站A需要有一个永久地址（home address），连接到归属网络（home network）。
  • 移动IP使用了代理，归属代理（home agent）通常就是连接在归属网络上的路由器。

课后习题

  • 为什么无线局域网中不能使用CSMA/CD协议而需要使用CSMA/CA协议
  1. 无法实现碰撞检测，由于无线网络的信号强度动态范围过大，接受信号的强度往往远远小于发送信号的强度，因此无线局域网的适配器无法进行碰撞检测。
  2. 存在隐蔽站和暴露站问题，当一个站点在发送数据的时候，并不能确认其他站点不会发送数据，因此很容易发生碰撞。
  3. 由于无线局域网无法进行碰撞检测，因此最好的办法是避免碰撞也就是CSMA/CA协议。
  • 为什么无线局域网的站点在发送数据帧时，即检测到信道空闲也仍然要等待一小段时间？为什么在发送数据帧的过程中不像以太网那样继续对信道进行检测？
  1. 比如说在收到RTS帧，收到CTS帧或者收到数据帧的时候，由于需要进行CRC检验，所以一定需要延迟一段小短时间，这一小段时间通常取SIFS。有时候发送数据，由于不知道其他站是否发送数据，则需要等待一个DIFS时间才能发送。
  2. 由于站点无法像以太网那样持续对信道进行检测，而且无法实现碰撞检测，也不能保证检测到的信道空闲就一定空闲，所以发送数据帧的过程中无法像以太网那样继续对信道进行检测。
  • 冻结退避计时器剩余时间的做法是为了使协议对所有站点更加公平，请进一步解释。 
  1. 冻结退避计时器就是退避计时器如果检测到信道空闲则减少时隙，如果检测到信道忙则冻结时隙，在检测到信道闲后等待一个DIFS继续计时。
  2. 这样可以避免在退避计时器时间减少到0时又遇到信道忙，这样会再次等待重新设置退避计时器，由于我们使用二进制退避算法，可以选择的时隙范围会再次变大非常大，可能会浪费很多时间。
  • 第四代蜂窝移动通信网与第三代蜂窝移动通信网的主要区别
  1. 第四代蜂窝移动通信的LTE标准，其取消了电路交换，所有数据交换都使用分组交换技术，也叫全网IP化。LTE标准的体系结构由三部分组成，用户设备UE，演进的无线接入网E-UTRAN（Evolved UTRAN）和演进的分组核心网EPC（Evolved Packet Core）。EPC上半部分是控制层面（传输信令），下半部分是用户层面（传输数据）。
  2. 且4G数据率比3G高，信道带宽大很多。
  • 为什么采用预约信道的方法可以较好地解决隐蔽站的问题？

如下图，假如其他站中有隐蔽站。

我们源站向AP发送完RTS后，这时隐蔽站不会接受到RTS，AP确认后会向源站返回一个CTS帧，这个帧会同时发送个其他所有站，包括隐蔽站，那么隐蔽站就会根据CTS帧中的A的预约时间建立NAV，知道这一段时间信道忙。

当然，这样的化在RTS发送时可能会由于隐蔽站问题产生碰撞，但是由于RTS帧很短，重新发送一次也不会浪费很多资源。

  • 有下图所示的四个站点使用同一无线频率通信，每个站点的无线电覆盖范围都是如图所示的椭圆形。也就是说，A 发送时，仅仅 B 能够接收；B 发送时，A 和 C 能够接收；C 发送时，B 和 D 能够接收；D 发送时，仅仅 C 能够接收。
    现假定每个站点都有无限多的报文要向每一个其他站点发送。若无法直接发送，则由中间的站点接收后再转发。例如，A 发送报文给 D 时，就必须经过 A 到 B，B 到 C 和 C 到 D 这样三次发送和转发。时间被划分成等长的时隙长度。在一个时隙中，一个站点可以做以下事情中的一个：① 发送一个报文；② 接受一个发给自己的报文；③ 什么也不做。
    再假定传输无差错，在无线电覆盖范围内都能正确接收。

    （1）假定有一个全能的控制器，能够命名各站点的发送或接受。试计算从 C 到 A 的最大数据报文传输速率（单位为报文/时隙）。
    （2）假定现在 A 向 B 发送报文，D 向 C 发送报文。试计算从 A 到 B 和从 D 到 C 的最大数据报文传输速率（单位为报文/时隙）。
    （3）假定现在 A 向 B 发送报文，C 向 D 发送报文。试计算从 A 到 B 和从 C 到 D 的最大数据报文传输速率（单位为报文/时隙）。
    （4）假定本题中所有无线链路都换为有线链路。重做以上的（1）至（3）小题。
    （5）现在再回到无线链路的情况。假定在每个目的站点收到报文都必须向源站点发回 ACK 报文，而 ACK 报文也要用掉一个时隙。重做以上的（1）至（3）小题。

  1. C->B，B->A，也就是1报文/2时隙。
  2. A->B和D->C是同时的，所以2报文/1时隙。
  3. 由于C发报文时B会检测到信道忙，所以先C->D，然后A->B，所以是1报文/1时隙。
  4. 有线链路
    1. 第一个报文1报文/2时隙，后面由于C->B和B->A可以同时进行，所以1报文/1时隙
    2. 同时进行，2报文/1时隙
    3. 同时进行，2报文/1时隙
  5. ACK报文
    1. 同无线，但是每次发送会返回一个ACK报文，所以1报文/4时隙
    2. 2报文/3时隙
      1. A->B，D->C
      2. B返ACK
      3. C返ACK
    3. 2报文/3时隙
      1. C->D
      2. D返ACK，A->B
      3. B返ACK
