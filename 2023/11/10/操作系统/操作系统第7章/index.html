操作系统第7章 输入输出系统

重要知识

知识点

I/O系统基本概念

基本功能

  1. 隐藏物理设备的细节；
  2. 与设备无关性；
  3. 提高处理机和I/O设备的利用率；
  4. 设备控制；
  5. 设备正确共享；
  6. 错误处理；

I/O系统层次（上四层）

I/O系统模型

I/O系统中的上下接口，上接口为I/O系统接口，下接口为软件/硬件接口，中间为I/O系统，也就是从设备独立性软件到中断程序间。

I/O系统的接口分为

1、块设备接口

数据传输和存取都以块为单位，速率较快；

可寻址：将磁盘的二级结构：磁道和扇区隐藏，映射为线性序列；

将抽象命令映射为底层操作，例如如果用户需要读磁盘，则将逻辑块号映射为物理盘面、磁道和扇区号；

2、流设备接口

数据传输和存取都以字符为单位；慢。

不可寻址，采用顺序存取方式，利用put和get操作；[回忆一下生产者和消费者问题]。

in/control操作，属于互斥设备，需要用打开和关闭操作实现互斥使用。

3、网络通信接口

I/O设备和设备控制器

I/O设备

I/O设备一般由两部分组成：

  1. 机械部分：执行I/O操作，例如键盘的机械部分，鼠标的机械部分

  2. 电子部分：也叫设备控制器或者适配器（adapter），在微机和小型机一般都是控制卡，接口卡或者网卡；在大中型计算机系统中，会配置I/O通道或者I/O处理机。

分类：

  1. 使用特性：存储设备、I/O设备。
  2. 传输速率：低速设备（鼠标键盘）、中速设备（打印机）、高速设备（磁带机、磁盘机）。

I/O设备与设备控制器的接口：

  1. 数据信号线
  2. 状态信号线
  3. 控制信号线

设备控制器

设备控制器基本功能

  1. 接收和识别命令
  2. 数据交换
  3. 标志和报告设备状态
  4. 地址识别
  5. 数据缓冲区
  6. 差错控制

设备控制器的组成

  1. 控制器与CPU的接口
    1. 数据信号线
    2. 地址信号线
    3. 控制信号线
  2. 设备控制器与设备的接口
    1. 数据信号线
    2. 控制信号线
    3. 状态信号线
  3. I/O逻辑
    1. I/O逻辑用于实现对设备的控制，
    2. CPU通过控制线向控制器发送I/O命令，I/O逻辑对收到的命令进行译码。

I/O通道

  1. 可以通过访问内存，取指执行。可以看作是一种特殊的处理机。
  2. 能够执行的指令类型单一；仅限于I/O指令。
  3. 没有自己的内存，和CPU共享内存。

在设置了通道后，CPU只需向通道发送一条I/O指令。通道在收到该指令后，便从内存中取出本次要执行的通道程序，然后执行该通道程序，仅当通道完成了规定的I/O任务后，才向CPU发中断信号。

I/O通道类型：

  1. 字节多路通道：主通道连接多个子通道，每个子通道连接一个设备，按时间片共享主通道(以字节为单位) ［只适合低速设备］
  2. 数组选择通道：主通道只含一个分配型子通道，一段时间只控制一台设备，按照数组单位传送；［适合高速设备，但利用率低］
  3. 数组多路通道：以上两种的结合

通道多通路，即一个通道可以连接多个控制器，控制器连接多个I/O设备。

寻址方法：内存映像I/O

在编制方式上不区分内存和设备控制器中的寄存器地址，进行统一编址，只不过范围不同，例如设定n，如果<=n，则是内存地址，>n则是设备地址；

中断和中断处理程序

中断处理程序是I/O系统中最低的一层，（上层是设备驱动程序，与设备无关的I/O软件）。

中断和陷入

  1. CPU对I/O设备发来的中断信号的一种响应。CPU此时会暂停当前执行的程序，保留环境，转入中断处理程序，结束后，再回到断点，继续执行原来的程序；这种中断是由外部设备引起的又称外中断；
  2. 陷入是指CPU内部异常事件，例如运算溢出，故障等发生后，CPU暂停正在执行的程序，转去执行该陷入事件的处理程序，这类也称为内中断。

中断向量表：通过中断序号找到中断处理程序入口。

中断优先级：嵌套中断。

处理多中断方式：

  1. 屏蔽（禁止）中断：顺序执行，执行一个中断屏蔽其他的。
  2. 嵌套中断：优先级决定。

中断处理程序

比如I/O过程，用户进程请求I/O后，OS先调用该设备的驱动程序，阻塞进程、执行完指令后，发出中断信号。

  1. 测定是否有未响应的中断信号。
  2. 保护现场，由硬件将CPU状态字（PSW）和程序计数器（PC）中的下一条指令的地址保存在中断栈，然后将寄存器等内容都压栈。
  3. 转入中断处理程序。处理机对信号源测试，找到中断设备号，并向其发出确认信号，然后将中断信号取消，继而，将设备中断处理程序的入口地址放入程序计数器中，开始执行中断处理程序。
  4. 中断处理。
  5. 恢复现场并退出中断。

设备驱动程序

中断是I/O结束的时用的，驱动则主要是I/O启动时用的。

功能

  1. 驱动程序是I/O系统的高层和设备控制器之间的通信程序，其需要接收抽象的I/O指令，例如 read 或者 write，然后将其转换为适合该硬件的具体指令和参数。
  2. 检查用户I/O的合法型，了解I/O设备的状态，设置设备的工作方式。
  3. 发出I/O命令，如果设备空，则启动设备，如果忙则阻塞，挂在该设备的等待队列上。
  4. 响应中断并转入中断处理程序。

和一般应用程序不同

  1. 仅转换I/O相关操作。
  2. 和硬件紧密关联，同类终端可以共享一个驱动。
  3. 驱动程序与I/O采用的控制方式相关，一般为中断驱动或者DMA方式。
  4. 和硬件紧密相关，因此一部分是汇编语言，固化在ROM中。
  5. 允许可重入，正在运行的驱动程序常会在一次调用完成前再次被调用。

处理过程

  1. 将抽象要求转换为具体要求，以磁盘为例，将逻辑地址转换为盘号、磁道号和扇区号。
  2. 对服务请求进行校验。检查用户请求的合理性，例如，从打印机读入数据是不合理的。
  3. 检查设备的状态，启动I/O设备前，该设备应该是就绪状态，需要检查设备控制器的状态寄存器的，如果不是，则阻塞进程到该设备的等待队列上。
  4. 传送参数（命令外的其他约束）。
  5. 启动I/O设备，传送指令。
  6. 将自己阻塞，等待设备I/O结束后的中断信号再被唤醒。

对I/O设备的控制方式
1、可轮询的可编程I/O方式

2、中断可编程

优点

3、DMA直接存储器方式

组成结构：主机与DMA控制器的接口、DMA控制器与块设备的接口、I/O控制逻辑。

主机与DMA控制器的接口

工作过程

①从磁盘读入数据块时，CPU将一条读指令送至磁盘控制器的命令寄存器(CR)中。

②将数据读入的内存起始目标地址送入内存地址寄存器（MAR）中；将读数据的字(节)数送入数据计数器(DC)中

③将磁盘的源地址送至DMA控制器的I/O逻辑中。

④然后，启动DMA控制器进行数据传送，以后，CPU便可去处理其它任务。

⑤此后，整个数据传送过程便由DMA控制器进行控制。

⑥当DMA控制器已从磁盘读入一个字(节)的数据并送入数据寄存器(DR)后，将该字(节)传送到指定的内存单元中。

⑦接着便对MAR内容加1，将DC内容减1。

⑧若减1后DC内容不为0，表示传送未完，便继续传送下一个字(节)。

⑨否则，由DMA控制器发出中断请求。

4、I/O通道方式

是DMA方式的发展，把对一个数据块的读写，改为对一组数据块的读写；

CPU、通道、设备同时并行；

当CPU要完成一组数据块读写时，只需要给I/O通道发一条指令，给出执行通道程序的首地址和设备号，通道接收后，可以自行访问内存地址并读取通道程序指令进行控制，直到所有指令执行完毕后才发中断。

通道程序

  1. 操作码：规定指令所执行的操作，如读/写/控制等。
  2. 内存地址：标明数据送入内存(读操作)和从内存取出(写操作)时的内存首址。
  3. 计数：表示指令所要读(或写) 的字节数。
  4. 通道程序结束位P：表示通道程序是否结束。P=1表示本指令是通道程序的最后一条指令。
  5. 记录结束标志R：R=0表示本指令与下一条指令所处理的数据是同属于一个记录；R=1表示这是处理某记录的最后一条指令。

与设备无关的I/O软件

引入了逻辑设备名（打印机）和物理设备名（得力打印机）。

功能

  1. 设备驱动程序的统一接口

  2. 缓冲管理

  3. 差错控制（暂时性和持久性）

  4. 对独立设备的分配与回收（有独立和共享设备）

  5. 独立于设备的逻辑数据块

设备分配与回收

物理设备名->SDT->DCT->COCT>CHCT

如独占设备：系统先从SDT中找到第一个类型的设备的DCT，先分配设备，再分配控制器，再分配通道。

设备控制表DCT

控制器控制表COCT

通道控制表CHCT

系统控制表SDT

分配设备考虑因素

  1. 设备固有属性，独占设备、共享设备、虚拟设备。
  2. 设备的分配算法，FCFS、最高优先级。
  3. 设备分配中的安全性，安全分配方式串行无死锁，不安全分配方式并行有死锁。

逻辑设备表：

  1. 整个系统设置一张。所有用户不能使用相同逻辑设备名，不适合多用户。
  2. 各个用户设置一张。用户登陆时便创建一个进程，同时创建一张逻辑设备表。

用户层的I/O软件

用户层的I/O软件：包括帮助用户进程实现系统调用的库函数和假脱机系统

系统调用和库函数

系统调用和库函数不一定一一对应，我们通过库函数来进行系统调用，比如很多现代OS调用都是通过c语言编写，而系统调用是我们取得OS服务的唯一途径。

假脱机系统

实现多个用户共享一台物理I/O设备。

我们可以看到，其实就是多了一个输入井，比如键盘输入字符，先输入到输入缓冲区，然后到输入井，用户从输入井中取数据。

输出也一样，用户将数据放到输出井，然后经过输出缓冲区进入I/O通道输出。

  1. 输入井和输出井：在硬盘上分出的空间进行数据存储，收容数据。
  2. 输入缓冲区和输出缓冲区：数据的中转站。
  3. 输入进程和输出进程：模拟两台外围机。比如输入过程中，用来将数据从输入设备传送到输入缓冲区，再放到输入井中。
  4. 井管理程序：直接和用户作业打交道。

优势：

  1. 提高I/O的速度；和输入井（高速I/O）打交道，而不是和低速I/O设备。
  2. 将独占设备改为共享设备。
  3. 实现了虚拟设备功能（貌似添加了多台设备）。

打印机假脱机

磁盘缓冲区（输出井）、打印缓冲区（输出缓冲区）、假脱机管理进程（井管理进程）和假脱机打印进程（输出进程）。

守护进程

将管理进程和脱机打印进程的功能又重新整合。

①为用户申请缓冲区并将缓冲区地址返回给用户进程后将自己阻塞；

②用户进程自己把文件放在缓冲区，并且把文件挂在打印机的打印队列上；

③唤醒守护进程，进行打印。

缓冲区管理

为什么引入缓冲区

  1. 缓解CPU和I/O设备速度不匹配的矛盾。

  2. 减少对CPU的中断频率；只有1bit缓冲区，那么每1bit数据都要中断CPU；缓冲区越大，中断次数越少。

  3. 解决数据粒度不匹配的问题；例如视频流是字符流的形式，但在播放的时候是以帧为单位的。

  4. 提高CPU和I/O设备的并行性。

单双缓冲区

单缓冲区：每当用户申请I/O的时候，OS就给该用户分配一个缓冲区。因此在CPU进行处理的时候（缓冲区数据已经送到CPU），CPU处理和用户输入是并行的。

双缓冲区：每次分配两个缓冲区，因此如果CPU处理速度小于输入速度，则可以一直输入，否则可以一直处理。

环形缓冲区

也就是设置多个缓冲区，形成环形，更好的并行运行。

缓冲池

缓冲池：将缓冲这种资源作为公共资源进行统一管理，不仅在内存中划分响应的存储区，而且还为其设计数据结构和调用函数（面向对象的思想）

组成：空白缓冲队列emq，输入队列inq，输出队列outq

工作方式

收容就是往缓冲池里放东西，提取就是拿出来。

  1. 收容输入：输入进程调用Getbuf(emq)，从emq队首拿出一段缓冲区，并把它作为收容输入工作缓冲区hin。装满后调用putbuf(inq, hin)，将它挂在输入队列inq上。
  2. 提取输入：Getbuf（inq）从输入队列取一个装满数据的输入缓冲区，作为sin；数据提取后再调用Putbuf（emq，sin）将空了的sin挂在emq上。
  3. 收容输出：Getbuf（emp）从空队列取一个空缓冲区，作为hout；数据收容后再调用Putbuf（outq，hout）将满了的hout挂在outq上。
  4. 提取输出：Getbuf（outq）从满的输出队列取一个数据缓冲区，作为sout；数据提取后再调用Putbuf（emq，sout）将空了的sout挂在emq上。

缓存只提供一个位于其他地方的数据项的更快存储副本。缓冲保留数据项的唯一现有版本。
