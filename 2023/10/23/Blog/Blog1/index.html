Hexo + Github Pages 搭建博客

发表于2023-10-23|更新于2023-10-23
|阅读量:

@TOC

所有资料都可以在这里找到：hexo中文官网

前提配置

注意：因为作者是一边做一边总结，所以有一些步骤并没有详细写出，比如github的创建，nodejs和git的安装，可能会补充，但其实在官网直接安装就可以啦！

  • Node.js 安装 ：提供npm指令
    成功后在cmd输入以下代码检验：

    node -v
    npm -v

  • Git 安装：用于部署Github
  • 一个github账号
  • Hexo在下文过程中安装，不需要先安装

Git绑定Github

注意git中复制粘贴是Ctrl + insert 和 Shift + insert

安装完git后，我们会有一个app为git bash，我们后续的操作基本都在这里进行，不要搞错了，打开如图：

检查SSH是否安装

在git bash输入：

ssh

在git bash复制公钥到Github

cd ~/.ssh
ls # 查看目录
cat id_rsa.pub

然后我们会看到一段文本，将其复制。
打开Github主页，点击右上角头像-settings-SSH and GPG keys-New SSH key，将复制的公钥粘贴到key内并创建。
(图中是已经完成的，点击New SSH key即可)

在git bash输入下文检验是否成功，出现”Hi 用户名！”字样即为成功。

ssh -T git@github.com

git连接Github仓库

1。 随便进入/新建一个自己的github repository，点击clone or download–clone with https，复制内容

  1. 然后进入一个新建的用来存储git仓库的目录，在git bash进入该目录（使用cd命令，如cd D:/git/repositroy）。

  2. 输入git clone + 刚复制的内容

    git clone github.com/…..

  3. 复制成功后，在里面新建一个text.txt文件，在此目录进入git bash，然后在git bash输入下面代码。
    三个代码分别是查看仓库状态，添加text.txt文件，提交信息

    git status
    git add text.txt
    git commit -m “commit text file”

  4. 如果是第一次提交，会让你绑定自己的github，先输入以下代码，“”里的内容是自己的用户名和github绑定的邮箱

    git config –global user.name”你的名字”
    git config –global user.email”@.com”

  5. 再输入下列命令，将本地仓库提交到远程仓库

    git push origin main

第一次上传需要输入github密码，通过github验证
7. 上传成功后，我们看修改的github的仓库，多了一个text.txt即为成功

设置npm的系统全局变量

  1. 在nodejs文件里新建两个空文件夹node_cache、node_global
  2. 打开cmd，输入以下命令，注意最后“”里的内容是你这两个文件的路径

    npm config set prefix “D:\nodejs\node_global”
    npm config set cache “D:\nodejs\node_cache”

  3. 设置环境变量
    打开环境变量：直接在windows搜索环境变量打开。

先在下面的系统变量一栏点击新建：变量值D:\nodejs\node_global\node_modules

然后再上面的用户变量里，选定path，点击编辑，新页面点击编辑，输入D:\nodejs\node_global
点击确定

  1. 在cmd命令行输入

    npm install webpack -g

设置github pages repository

在github新建一个repository，仓库命名为用户名.github.io，后续我们将网站部署到这个仓库上

安装并部署Hexo

在D盘新建一个文件夹Blog。==打开git bash并进入该目录==

  1. Hexo 安装：

    npm install hexo-cli -g

  2. 安装完成后，输入hexo init进行初始化

  3. 然后输入hexo g进行静态部署，可以使用hexo s命令查看是否部署完成。hexo s查看过程中可以输入 http://localhost:4000 查看部署的网页，点击Ctrl + C退出查看。

  4. 我们先查看我们github pages仓库的ssh链接，复制
    地址：你刚刚创建的github pages仓库-code Clone-SSH

  5. 回到我们的Blog文件夹，用任意文本编辑器打开_config.yml文件，下滑到底部，将最下方的deploy内容修改为如下形式：

    deploy:
    type: git
    repository: git@github.com:etherea1ity/etherea1ity.github.io.git # 刚刚复制的链接
    branch: master

  6. 在git bash进入Blog文件夹，安装Git部署插件

    npm install hexo-deployer-git –save

  7. 输入下列三条命令

    hexo clean # 清除缓存
    hexo g # 生成网站静态文件到默认public文件夹
    hexo d # 生成网站静态文件并部署

  8. 完成后打开浏览器，输入https://xxx.github.io就可以打开了

修改域名

虽然我们已经部署好了hexo网页，但是域名只能使用github的域名，那么如何修改呢？

购买域名

在阿里云万网上购买域名，可以是.com，.top等结尾，一般来说，top比较便宜，com比较贵。
我们输入想设定的域名进行查询，进行购买。过程中需要实名认证，需要提供身份证证件正面并通过审核，该审核过程大概半个小时。

解析域名

  1. 登录阿里云万网，进入域名控制台，我是一般在控制台-最近-域名中进入的

  2. 在刚刚购买的域名后面点击解析，按照下文配置，先配置域名。（这里点击的是新手引导配置选项）

  3. 配置IPV4地址。
    打开cmd输入（ping -4+ github pages地址），这里-4代表IPV4，-6代表IPV6

    ping -4 etherea1ity.github.io

然后继续在域名控制台配置，注意这次选择的是IPV4**@** 的那个配置。
配置完后如下图

  1. 打开Blog文件夹中的source文件夹，添加一个CNAME文件，我们可以先创建CNAME.txt文件，在里面输入你的网页名，然后==再删除txt文件后缀==。这里如果加上www.的话每次进入网站都必须输入www.才能进入。如图

如果没有办法修改后缀，在这里显示后缀：

  1. 回到Blog文件夹，使用git bash进入Blog，输入下面三条命令

    hexo clean
    hexo g
    hexo d

  2. 进入github pages仓库，在设置中找到custom domain，输入域名并且保存

  3. 到这一步为止，我们就已经可以通过自己的域名登上网站啦！
